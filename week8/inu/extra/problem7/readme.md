# 문제 개요

문제 : [궁금한 민호](https://www.acmicpc.net/problem/1507)

문제 분류 : 플로이드-와샬

# 문제 풀이

플로이드 알고리즘의 기본 개념은 두 정점 u, v 사이의 거리가 다른 정점 k를 거쳐서 가는 거리보다 크다면 k를 거쳐서 가는 경로로 업데이트를 하는 것이다.

이 개념을 문제에 적용시켜보자.

1. 주어진 입력은 이미 두 도시 사이의 최소 이동 시간이기 때문에 경로를 업데이트 할 때 다른 도시를 거쳐서 갈 때 더 적은 시간이 걸린다면 입력부터 잘못되었기 때문에 -1을 출력해야 한다.

2. 두 도시 사이의 이동 시간이 다른 도시를 거쳐서 가는 시간과 똑같으면 두 도시 사이에 있는 도로는 없어도 되기 때문에 이동 시간을 INF로 설정한다.

주의할 점은 서로 다른 3개의 다리를 비교해야 하기 때문에 플로이드의 3중 for문에서 k, u, v의 값이 모두 다를 때만 이동 시간을 비교해야 한다.  
그리고 1번의 경우 두 도시 사이의 다리가 없어져서 값이 INF인 경우에는 -1을 출력하지 않아야 한다.
